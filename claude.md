# Claude Instructions for This Project

## Project Purpose

This project is built and maintained primarily with AI assistance.
The goal is to keep the app simple, stable, and easy to improve over time.

## Core Rules (IMPORTANT)

* Do NOT remove existing features unless explicitly asked
* Do NOT refactor large sections of working code
* Prefer small, safe edits over big rewrites
* Keep the UI clean and simple
* Avoid adding unnecessary complexity
* If unsure, preserve current behavior

## Coding Style

* Keep files readable and organized
* Use clear variable names
* Avoid over-engineering
* Comment important logic in plain English

## When Adding Features

* Add features in the simplest way possible
* Do not change unrelated parts of the app
* Maintain current layout and structure
* Make UI improvements subtle and clean

## When Fixing Bugs

* Fix only what is broken
* Do not rewrite entire systems to fix small issues
* Keep the original design intact

## UI Guidelines

* Prioritize neat, minimal design
* Improve spacing, alignment, and clarity over adding new elements
* Do not redesign the whole interface unless asked

## Safety Rules

* Never delete user data
* Never remove folders or stored content structures
* Avoid database/schema changes unless explicitly requested

## Project Context

This app is built by someone with little coding knowledge using AI tools.
Changes should be:

* Easy to understand
* Easy to maintain
* Low risk

## Communication Style

When explaining changes:

* Be simple
* Be direct
* Avoid technical jargon
* Explain what changed and why
